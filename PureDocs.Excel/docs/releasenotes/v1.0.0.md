## Tổng quan

**TVE.PureDocs.Excel** là thư viện .NET thuần túy để tạo, đọc, ghi và tính toán tệp Excel `.xlsx` không cần cài đặt Microsoft Office. Tất cả thao tác thực hiện hoàn toàn trong bộ nhớ, hỗ trợ lưu ra file hoặc `Stream`.

---

## 1. Quản lý Workbook

| Tính năng | Mô tả |
|-----------|-------|
| Tạo workbook mới | `Workbook.Create()` — tạo workbook trống với stylesheet mặc định |
| Mở từ file | `Workbook.Open(string filePath)` |
| Mở từ stream | `Workbook.Open(Stream stream)` — stream gốc không bị ảnh hưởng |

| Flush stylesheet lazy | Stylesheet được ghi một lần duy nhất khi `SaveAs`, không ghi từng thao tác |

## 5. Công thức (Formula Engine)

### Gán và đọc công thức
| Tính năng | Mô tả |
|-----------|-------|
| Gán công thức | `cell.SetFormula("SUM(A1:A10)")` — tự bỏ dấu `=` đầu |
| Đọc công thức | `cell.GetFormula()` → `string?` |
| Kiểm tra có công thức | `cell.HasFormula` → `bool` |
| Tính toán tất cả công thức | `sheet.CalculateFormulas()` |
| Tính toán thông minh | `sheet.SmartRecalculate()` — chỉ tính ô bị ảnh hưởng; trả về số ô đã tính |
| Đánh giá công thức tức thời | `sheet.EvaluateFormula("SUM(A1:A5)")` |

### Cơ chế recalculation
- **Dependency Graph**: Mỗi công thức được phân tích thành cây phụ thuộc. Khi một ô thay đổi, chỉ các ô phụ thuộc vào nó mới được tính lại.
- **Topo sort**: Thứ tự tính toán đảm bảo ô nguồn được tính trước ô phụ thuộc.
- **Phát hiện vòng lặp**: Các ô trong chu trình nhận giá trị `#REF!`.
- **Hàm volatile**: `RAND`, `RANDBETWEEN`, `NOW`, `TODAY`, `OFFSET` luôn được tính lại.
- **Tính toán song song**: `CalcChain.ParallelRecalculate()` — các ô độc lập tính đồng thời.
- **LRU AST Cache**: Công thức được parse 1 lần, AST tái sử dụng (mặc định 10.000 entry).

### Hàm toán học (`MathFunctions`)

### Hàm logic (`LogicalFunctions`)
`IF`, `AND`, `OR`, `NOT`, `XOR`, `IFERROR`, `IFNA`, `IFS`, `SWITCH`, `TRUE`, `FALSE`

### Hàm ngày/giờ (`DateFunctions`)

_\* volatile_

### Tính năng nâng cao của Formula Engine
| Tính năng | Mô tả |
|-----------|-------|
| Named Ranges | `NamedRangeManager` — đặt tên cho vùng ô (phạm vi workbook hoặc sheet) |
| Structured References | `TableManager` — tham chiếu bảng kiểu `Table1[Column1]`, `[#Headers]`, `[#Totals]`, `[#All]`, `@Column` |
| 3D References | `ThreeDReference` — tham chiếu dải sheet kiểu `Sheet1:Sheet3!A1` |
| Implicit Intersection | Toán tử `@` — Excel 365 implicit intersection |
| Cross-sheet references | `SheetName!CellRef` và `'Sheet Name'!CellRef` |
| Culture-aware parsing | `CultureCoercion` — phân tích số/ngày theo `InvariantCulture` hoặc `CurrentCulture` |
| Dynamic dependencies | `DynamicEvaluator` — xử lý `INDIRECT`/`OFFSET` có dependency runtime, giới hạn đệ quy 100 cấp |
| Lazy range iterator | `LazyRangeIterator` — duyệt vùng ô lớn mà không tạo toàn bộ mảng trong bộ nhớ |
| Array Pool | `FormulaValuePool` — tái sử dụng mảng tạm qua `ArrayPool<T>` |
| Cell reference interning | `CellReferenceIntern` — cache chuỗi địa chỉ ô (giảm allocation) |

## 9. Hiệu năng và nội bộ

| Tính năng | Mô tả |
|-----------|-------|
| SharedStringTable | Thread-safe; O(1) tra cứu 2 chiều; giảm dung lượng file khi text lặp lại |
| LRU AST Cache | Mặc định 10.000 entry; cấu hình qua `FormulaEvaluator.MaxCacheCapacity` |
| Style deduplication | 5 caches nội bộ (font/fill/border/numberFormat/cellFormat) |
| RangeSet | `SUM(A:A)` lưu 1 `RangeArea` thay vì 1.048.576 địa chỉ ô riêng lẻ |
| Array Pool | Tái sử dụng `FormulaValue[]` và `double[]` tạm qua `ArrayPool<T>` |
| Parallel recalculation | `ParallelRecalculate()` — tận dụng đa luồng cho workbook lớn |
| LazyRangeIterator | Duyệt vùng ô lớn mà không cần tạo toàn bộ mảng trong bộ nhớ |
| Cell reference intern | Cache chuỗi địa chỉ ô để giảm GC pressure |
| Performance counters | `FormulaPerformanceCounters` — theo dõi số lần parse, eval, cache hit/miss, thời gian recalc |
